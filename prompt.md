# MNIST-C 项目需求

## 核心需求

用纯 C 语言实现 MNIST 手写数字识别系统，不依赖任何第三方库，仅使用标准 C 库。仅在 x86 系统下完成所有任务。

## 功能需求

### 1. 神经网络实现

- 3 层全连接网络：784 → 128 → 10
- 隐藏层激活函数：ReLU
- 输出层激活函数：Softmax
- 损失函数：交叉熵
- 优化器：小批量 SGD
- 权重初始化：He 初始化（Box-Muller 正态分布 × √(2/fan_in)）
- 目标准确率：5 个 epoch 达到 97% 以上

### 2. 目录结构规范

- 源代码放在 `src/` 目录下
- 编译中间文件（.o）输出到 `build/` 目录
- 可执行文件输出到 `bin/` 目录
- MNIST 数据集放在 `data/` 目录下
- 根目录只保留 Makefile、README 等功能入口文件

### 3. MNIST 数据集下载功能

- 提供数据集下载脚本或工具，自动下载 MNIST 数据集
- 数据集包含训练集（train-images-idx3-ubyte, train-labels-idx1-ubyte）和测试集（t10k-images-idx3-ubyte, t10k-labels-idx1-ubyte）
- 下载的数据集应保存到 `data/` 目录
- **中国大陆可访问的下载方式**：使用国内镜像源（如清华镜像源、阿里云镜像源）或 GitHub 镜像站下载数据集
- **备用方案**：如果主要下载源不可用，应提供备用下载源
- **下载工具选择**：优先使用 `wget` 或 `curl`，确保在标准 Linux 环境中可用
- **完整性验证**：下载后应验证文件完整性（如检查文件大小或使用校验和）
- 支持离线使用：如果 `data/` 目录已存在数据集文件，则跳过下载
- 下载功能应集成到 Makefile 中，通过 `make data` 命令触发
